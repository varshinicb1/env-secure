🔍 Scan your codebase for leaked secrets before they become a headline

env-secure is a standalone CLI that scans your entire codebase for API keys, passwords, tokens, and other secrets — then generates a hardened .env.example and an encrypted vault so you never commit credentials to git again.

One command. Instant security audit. Zero false positives engineered.

⚡ Features

  • 60+ detection patterns — curated regex library for every major service
  • Entropy analysis — Shannon entropy filtering eliminates false positives
  • Gitignore-aware — automatically skips node_modules, .git, build artifacts
  • Binary-safe — doesn't scan images, videos, archives, or compiled files
  • Smart dedup — nearby matches merged, highest-confidence result kept
  • Secret masking — all output safely masked (shows first 6 chars only)
  • Multiple output formats — terminal, JSON, GitHub Actions markdown
  • CI/CD ready — auto-detects CI environments, works without TTY
  • .env.example generator — extracts env var names, generates ready-to-fill template
  • Encrypted vault — AES-256-GCM encrypts discovered secrets with your passphrase
  • Merge mode — combines scan results with existing .env.example files

💻 Quick start

# Scan current directory
env-secure

# Scan specific directory with JSON output
env-secure --dir ./my-project --json

# Generate .env.example from discovered secrets
env-secure --env

# Create encrypted vault
env-secure --vault

# Full pipeline: scan + env + vault
env-secure --env --vault

# CI mode (auto-detected, no prompts)
env-secure --ci

# GitHub Actions compatible output
env-secure --ci-matrix

❓ FAQ

How is this different from git-secrets or truffleHog?
env-secure is a single static binary (no dependencies), has 60+ curated patterns, generates .env.example files, and creates an encrypted vault — all in one tool.

Does it upload my code anywhere?
No. All scanning is done locally. Nothing leaves your machine.

What about false positives?
We use entropy analysis, pattern-specific filtering, and smart deduplication to minimize false positives. You can adjust the entropy threshold with --entropy-threshold.

Can I add custom patterns?
The patterns library is open-source and extensible. The source code is included.
